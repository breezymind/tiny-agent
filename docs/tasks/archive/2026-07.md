### T-010 [ready-for-agent] issue-store CRUD와 벡터 검색 CLI 완성

**Status:** done

## Parent

- T-006

## What to build

`scripts/issue-store.js`에 `create`, `get`, `list`, `update-status`, `search` 명령을 완성한다. `create`는 `T-###` 번호를 단조 증가로 발급하고 PRD와 일반 이슈를 모두 같은 이슈/문서 모델로 저장한다. `list`와 `get`은 to-prd/to-issues 및 코딩 에이전트가 사용할 수 있는 안정적인 JSON을 출력한다. `search`는 query를 임베딩한 뒤 sqlite-vector의 `vector_full_scan`을 사용하고 `embedding_status=ready` 문서만 대상으로 한다.

## Acceptance criteria

- [ ] `create`가 새 `T-###` issue_id를 발급하고 label/status/parent/blockedBy/body를 저장한다.
- [ ] PRD는 별도 엔티티가 아니라 `(PRD)` 부모 이슈의 문서 행으로 저장된다.
- [ ] `get`이 단일 이슈의 메타데이터, 관계, 본문, 임베딩 상태를 JSON으로 출력한다.
- [ ] `list`가 status/label/parent 필터로 이슈 목록을 JSON으로 출력한다.
- [ ] `update-status`가 backlog/current/done 상태 전이를 DB에 반영한다.
- [ ] `search`가 ready 임베딩만 대상으로 nearest 문서를 반환하고 rowid 결과를 이슈 메타데이터와 조인한다.
- [ ] CLI seam 테스트가 각 명령의 외부 동작을 검증한다.
